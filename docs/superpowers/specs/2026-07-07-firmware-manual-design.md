## 1. ภาพรวมและเป้าหมาย (Overview & Goals)

คู่มือฉบับนี้มีวัตถุประสงค์เพื่อเป็นคู่มือประจำอุปกรณ์ (Product Manual) สำหรับผู้ติดตั้งฮาร์ดแวร์ (Installer) และผู้ใช้งานทั่วไป (Operator/Owner) โดยถูกจัดเก็บในรูปของ **Single-File HTML** ภายในโฟลเดอร์ `docs/manual.html` เพื่อให้สามารถคัดลอกไปใช้อ่านแบบออฟไลน์ผ่านคอมพิวเตอร์ แท็บเล็ต หรือสมาร์ทโฟนได้โดยไม่ต้องมีสัญญาณอินเทอร์เน็ต

เอกสารนี้จะออกแบบตามหลักการ **"The Clear Console"** ซึ่งใช้โทนสีและโครงสร้าง UI ที่เข้าคู่กับ Web Dashboard ของ ESP32 เพื่อสร้างความต่อเนื่องทางแบรนด์ (Brand Identity) และลดความยุ่งยากในการอ่านคู่มือด้วยการฝังรูปภาพไดอะแกรมแบบ SVG ไดนามิกในไฟล์โดยตรง

## 2. โครงสร้างและการจัดเลย์เอาต์หน้าคู่มือ (Layout & Design System)

คู่มือจะถูกเขียนด้วยภาษา HTML5 และจัดแต่งสไตล์ด้วย CSS ในตัว (Embedded Stylesheet) โดยไม่มีการเรียกใช้ CDN หรือไฟล์สไตล์ภายนอก

### ธีมสี (Colors & Theme)
*   **Default Theme:** Dark Mode (เพื่อให้อ่านได้สบายตาในการทำงานภาคสนามหรือกลางคืน) และรองรับการสลับเป็น Light Mode ผ่านปุ่มด้านบน
*   **ระบบสีตามสถานะการทำงาน (Operational Semantics):**
    *   `--primary`: `#0097a7` (Teal) สำหรับโครงสร้างนำทาง หัวข้อ และปุ่มแอคชัน
    *   `--success`: `#0e9a5c` (Green) สำหรับสถานะปกติและปุ่มเริ่มทำงาน
    *   `--warning`: `#c47d00` (Amber) สำหรับข้อควรระวัง
    *   `--danger`: `#c53030` (Red) สำหรับจุดแจ้งเตือนวิกฤตที่อาจทำให้อุปกรณ์เสียหาย
    *   `--bg-body`: `#08090d` (Dark Background) / `#f0f2f7` (Light Background)

### โครงสร้างการแสดงผล (Visual Hierarchy)
*   **Responsive Desktop Layout:** ใช้ระบบ Split Sidebar (ซ้ายเป็นเมนูนำทางขยับตามการเลื่อนหน้า - Sticky Navigation, ขวาเป็นเนื้อหาหลักกว้างสูงสุด `75ch` เพื่อให้อ่านง่าย)
*   **Responsive Mobile Layout:** Sidebar จะซ่อนอัตโนมัติและแสดงเป็นเมนูแบบเลื่อน (Drawer) หรืออยู่ด้านบนสุดสำหรับหน้าจอมือถือ
*   **การจัดการงานพิมพ์ (Print Layout):** การกำหนด `@media print` เพื่อซ่อนปุ่มนำทาง เมนูด้านข้าง และส่วนตกแต่ง เพื่อให้สามารถสั่งพิมพ์ลงกระดาษขนาด A4 ได้สวยงามและไม่มีหน้าเว้นว่างที่ไม่มีข้อมูล

## 3. สารบัญเนื้อหาอย่างละเอียด (Document Content Outline)

เอกสารจะแบ่งเนื้อหาออกเป็น 7 ส่วนหลัก เพื่ออธิบายทั้งงานประกอบฮาร์ดแวร์ การจัดการซอฟต์แวร์ และการใช้งานระบบ:

### ส่วนที่ 1: ภาพรวมระบบและหลักการสลับลอจิก (System Overview & Control Logic)
*   คำอธิบายการควบคุมปั๊มน้ำด้วยตัวควบคุมรีเลย์ 2 ตัวตามระดับน้ำจากลูกลอย
*   **Control Matrix:**
    *   **ลูกลอยจม/OFF (น้ำลด):** ระบบจะสลับไปทำงานบน **Timer 1** และขับเคลื่อนรีเลย์ 1 (Pump Relay 1) ตามรอบเฟส ON/OFF
    *   **ลูกลอยลอย/ON (น้ำเต็ม):** ระบบจะสลับไปทำงานบน **Timer 2** และขับเคลื่อนรีเลย์ 2 (Pump Relay 2) ตามรอบเฟส ON/OFF
*   *เป้าหมายเพื่อความปลอดภัย:* ปั๊มจะไม่สลับการทำงานกะทันหันหากลูกลอยมีอาการกระเพื่อมถี่ ๆ (มีระบบป้องกันสัญญาณรบกวนด้วย Software Debouncing)

### ส่วนที่ 2: คู่มือการติดตั้งฮาร์ดแวร์และการป้องกันความผิดพลาด (Hardware Wiring & Safety Guide)
*   *จุดเน้นวิกฤตเพื่อป้องกันความผิดพลาด:*
    1.  **ตารางพิน GPIO ต้องห้าม:** ห้ามใช้ GPIO 34-39 สำหรับส่งสัญญาณควบคุมรีเลย์ หรือพินแฟลชภายใน/PSRAM เพื่อไม่ให้บอร์ดบูตลูปหรือทำงานไม่ได้
    2.  **การติดตั้งเซนเซอร์วัดอุณหภูมิ DS18B20:**
        *   ต้องต่อตัวต้านทานคร่อมสัญญาณ (Pull-up Resistor) ขนาด `4.7 kOhm` ระหว่างขาข้อมูล (DQ) กับขาไฟเลี้ยง (3.3V) เสมอ
        *   **คำเตือนวิกฤต:** ห้ามต่อเซนเซอร์กับไฟ 5V เด็ดขาด เนื่องจากระดับตรรกะ (Logic Level) ของ ESP32 ทำงานที่ 3.3V การจ่ายไฟ 5V เข้าพินข้อมูลจะทำลายตัวประมวลผลทันที
    3.  **การต่อรีเลย์ควบคุมกำลัง (Relay Contacts):**
        *   แนะนำการใช้ Relay Module แยกแหล่งจ่ายไฟ (Optocoupler Isolation) เพื่อตัดสัญญาณรบกวนแม่เหล็กไฟฟ้า (EMI) จากขดลวดปั๊มน้ำ
        *   การเลือกการทำงานของขั้วรีเลย์ (Relay Polarity) ให้ตรงกับประเภทโมดูลที่ใช้ (Active-Low / Active-High)

### ส่วนที่ 3: แผนผังวงจรไฟฟ้าในรูปแบบ SVG (Visual Wiring Diagrams)
*   **Diagram 1:** แผนภาพขั้วต่อขาและไฟเลี้ยงของ ESP32 DevKit V1 30-pin และ ESP32-S3 DevKitC-1
*   **Diagram 2:** แผนภาพวงจรการเชื่อมต่อ DS18B20 และตัวต้านทาน 4.7 kOhm แบบละเอียด
*   **Diagram 3:** แผนภาพวงจรการเชื่อมต่อสวิตช์ลูกลอยแบบแห้งเข้ากับพินกราวด์ (GND)
*   **Diagram 4:** แผนภาพวงจรรีเลย์ควบคุมปั๊มน้ำ 1 และ 2 แยกฝั่งไฟควบคุมกระแสต่ำกับไฟจ่ายปั๊มกระแสสูง

### ส่วนที่ 4: การจัดการซอฟต์แวร์และคอมไพล์เฟิร์มแวร์ (Build & Flashing Guide)
*   ข้อกำหนดสภาพแวดล้อมระบบ: การติดตั้ง ESP-IDF v6.0.1 (ไม่รองรับ Arduino)
*   การรันคำสั่งจัดการระบบผ่าน Windows PowerShell:
    *   คำสั่งตั้งค่าหน้ากระดาษรหัสภาษาเพื่อป้องกัน Warning เรื่องอักขระ: `chcp 65001`
    *   การรันคำสั่งบิลด์และแฟลชแยกโปรไฟล์บอร์ด: `.\scripts\build.ps1 -Target esp32` หรือ `-Target esp32s3`
    *   การเรียกหน้าจอมอนิเตอร์แบบ Interactive: `.\scripts\flash_and_show.ps1`

### ส่วนที่ 5: การเริ่มต้นทำงานครั้งแรกและการตั้งค่า Wi-Fi (First-Time Connection Setup)
*   **ขั้นตอนการเข้าถึงบอร์ด:** บอร์ดจะทำงานในโหมด APSTA โดยสร้างเครือข่ายตั้งค่าชื่อ `FishPump-Setup`
*   **ระบบรหัสผ่านความปลอดภัยสูง:** รหัสผ่านตั้งค่าจะถูกสร้างแบบไดนามิกจากที่อยู่ MAC ประจำตัวของชิปแต่ละตัวแล้วนำมาทำ XOR Salt (เช่น รหัสผ่าน `AABBCCDD`) โดยจะแสดงที่หน้าจอ TFT หรือพอร์ต Serial ตอนบูตระบบครั้งแรก
*   **การเข้าสู่ระบบ:** การเข้าใช้งานหน้าเว็บผ่าน `http://192.168.4.1` หรือ `http://fishpump.local` (ผ่าน mDNS)
*   **บัญชีเริ่มต้น:** Username `admin` และ Password `admin123` พร้อมข้อบังคับการเปลี่ยนรหัสผ่านในครั้งแรกเพื่อความปลอดภัย

### ส่วนที่ 6: คู่มือการใช้งานหน้าเว็บ Dashboard และการปรับแต่งค่าผ่านระบบ (Dashboard Configuration Guide)
*   การเปิดใช้งานการหมุนเวียนปั๊มน้ำและลอจิกการหน่วงเวลาการเริ่มปั๊ม (Auto-Start config)
*   การเปลี่ยนพิน GPIO ผ่านระบบแบบปลอดภัย: การสลับการตั้งค่าแบบ Pending ซึ่งจะมีผลหลังบูตระบบใหม่เท่านั้นเพื่อป้องกันบอร์ดล็อกตัวเองขณะกำลังตั้งค่า
*   การควบคุมการทำงานของเครื่องลดอุณหภูมิ (Cooling Control Mode):
    *   **Auto:** เปิด/ปิดตามอุณหภูมิเซนเซอร์เมื่อเกินจุด Threshold + Hysteresis
    *   **Force Off:** สั่งปิดพัดลมระบายอากาศถาวร
    *   **Test On:** สั่งเริ่มระบบคูลลิ่งทดสอบเป็นเวลาจำกัดชั่วคราว
*   การจัดการอายุหน้าจอ TFT: ฟังก์ชันการปรับระดับความสว่างหน้าจอแสดงผล และโหมดพักหน้าจออัตโนมัติ (TFT Auto-Dim Timer 5 นาที) เพื่อรักษาอายุการใช้งานหน้าจอ

### ส่วนที่ 7: ตารางรหัสข้อผิดพลาดและการวิเคราะห์ปัญหา (Troubleshooting & Diagnostics)
*   รายการรหัสข้อผิดพลาดของเซนเซอร์อุณหภูมิ (DS18B20 Error Codes)
*   การสังเกตสถานะไฟแสดงสถานะหรือข้อความแจ้งเตือนบนตัวเก็บประจุไฟ/หน้าจอแสดงผล
*   ขั้นตอนการรีเซ็ตระบบกลับสู่สถานะโรงงานผ่าน Wi-Fi AP Recovery Mode

## 4. องค์ประกอบเชิงโต้ตอบเพิ่มเติมในคู่มือ (Interactive Enhancements)

เพื่อให้คู่มือนี้มีความทันสมัยและตอบสนองได้ดีกว่าคู่มือทั่วไป:
1.  **Interactive Pre-Power Checklist:** ช่างติดตั้งสามารถคลิกเครื่องหมายติ๊กถูก (Checkbox) ในขั้นตอนการเดินสายไฟทีละข้อ โดยหน้าจอจะคำนวณและแสดงเปอร์เซ็นต์ความพร้อมก่อนเปิดสวิตช์จ่ายไฟจริง เพื่อบังคับให้มีการตรวจสอบพิน GPIO และวงจรดึงกระแสซ้ำ
2.  **Code Copy Helper:** เพิ่มปุ่มคัดลอก (Copy Button) ข้างคำสั่ง Command Line หรือคำสั่ง PowerShell ในคู่มือบิลด์ เพื่อให้นำไปรันต่อได้ทันทีโดยไม่ต้องลากคลุมคัดลอกเอง
3.  **TOC ScrollSpy:** การลิงก์ขยับ Sidebar เมนูด้านข้างตามการสไลด์หน้าเอกสาร เพื่อให้รู้ว่ากำลังอยู่ในส่วนใดของเนื้อหา

## 5. การตรวจสอบความถูกต้อง (Verification Plan)

หลังสร้างหน้าเว็บ `docs/manual.html` จะมีการตรวจสอบดังนี้:
1.  **Offline Run Test:** ดับการเชื่อมต่ออินเทอร์เน็ตบนอุปกรณ์ทดสอบทั้งหมด แล้วเปิดหน้าคู่มือ เพื่อตรวจสอบว่าองค์ประกอบ รูปภาพ สไตล์ และฟังก์ชันสคริปต์โต้ตอบทุกส่วนยังคงโหลดและใช้งานได้ปกติ
2.  **Responsive Layout Check:** ตรวจสอบความถูกต้องของตารางและเมนูสารบัญในสมาร์ทโฟน แท็บเล็ต และคอมพิวเตอร์ผ่าน Browser Developer Tools
3.  **Contrast & Accessibility Verification:** ตรวจสอบอัตราส่วนความต่างสี (Color Contrast Ratio) ในโหมดมืดและสว่างของเนื้อหาทั่วไปว่าสูงกว่า `4.5:1` เสมอ
4.  **Print Preview Audit:** ตรวจสอบการสั่งพิมพ์หน้ารายการคู่มือโดยสั่งแปลงเป็น PDF เพื่อดูว่ามีเนื้อหาหรือกราฟิก SVG ตัดกันหรือหลุดขอบกระดาษหน้าตัดต่อหรือไม่
