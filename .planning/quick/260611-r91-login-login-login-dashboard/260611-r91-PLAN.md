<task type="auto">
  <name>Task 1: Server-side redirect logic in web_server.c</name>
  <files>main/web_server.c</files>
  <action>
    Modify `handle_get_login` in `main/web_server.c` to:
    1. Extract the session token from cookie headers using `get_session_from_request()`.
    2. Get the client IP using `get_client_ip()`.
    3. Validate the session using `session_validate()`.
    4. If valid, set headers (no-store, redirect to /dashboard via 302 Found status) and send response.
    5. If invalid or missing, continue serving index.html as before.
  </action>
  <verify>
    <automated>powershell.exe -ExecutionPolicy Bypass -File .\scripts\build.ps1</automated>
  </verify>
  <done>
    Authenticated requests to /login return 302 redirect to /dashboard, while unauthenticated requests serve index.html.
  </done>
</task>

<task type="auto">
  <name>Task 2: Client-side redirect in app.js SPA router</name>
  <files>main/static/app.js</files>
  <action>
    Modify `handleRoute()` in `main/static/app.js`:

  </done>
