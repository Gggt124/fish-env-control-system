<task type="auto">
  <name>Task 2: Stateless JWT Generation</name>
  <files>components/session/session.h, components/session/session.c</files>
  <action>
    Rewrite `session_create()`. Format a JWT header `{"alg":"HS256","typ":"JWT"}` and payload `{"sub":"admin","ip":"<client_ip>"}`.
    Base64Url-encode both. Concatenate as `header.payload`.
    Compute the HMAC-SHA256 signature of `header.payload` using the cached 32-byte secret via `mbedtls_md_hmac`.
    Base64Url-encode the signature and append to create `header.payload.signature` into `token_out`.
    Do NOT store the token in RAM. It must be completely stateless (AUTH-02, AUTH-03).
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>session_create outputs a valid IP-bound JWT signed with the NVS secret with no server-side state.</done>
</task>

<task type="auto">
  <name>Task 3: JWT Validation, IP Binding, and Logout</name>
  <files>components/session/session.h, components/session/session.c</files>
  <action>
    Change `session_validate` signature to take `const char* client_ip`.
    Rewrite `session_validate()` to: 1) Parse the given token string into header, payload, and signature. 2) Re-calculate HMAC-SHA256 signature for `header.payload` using the cached RAM secret. 3) Compare signatures to verify integrity. 4) Verify that the `ip` in the payload matches `client_ip` (AUTH-04).
    Since tokens are stateless, `session_destroy()` can be removed entirely or changed to a no-op, as actual logout will be handled client-side by clearing the cookie.
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>session_validate verifies JWT signatures and client IP purely statelessly.</done>
