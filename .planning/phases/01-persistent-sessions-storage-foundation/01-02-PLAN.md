<task type="auto">
  <name>Task 2: Stateless JWT Generation</name>
  <files>components/session/session.c</files>
  <action>
    Rewrite `session_create()`. Instead of storing in a RAM array, format a JWT header `{"alg":"HS256","typ":"JWT"}` and payload `{"sub":"admin"}` (or use the provided username). 
    Base64Url-encode both. Concatenate as `header.payload`.
    Compute the HMAC-SHA256 signature of `header.payload` using the cached 32-byte secret via `mbedtls_md_hmac`.
    Base64Url-encode the signature and append to create `header.payload.signature` into `token_out`.
    Remove the `s_sessions` array completely per user decision D-02 (cancelling AUTH-02's token ring buffer limitation).
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>session_create outputs a valid JWT signed with the NVS secret.</done>
</task>

<task type="auto">
  <name>Task 3: JWT Signature Validation</name>
  <files>components/session/session.h, components/session/session.c</files>
  <action>
    Rewrite `session_validate()`. Parse the given token string into header, payload, and signature components.
    Re-calculate the HMAC-SHA256 signature for the `header.payload` portion using the RAM-cached secret.
    Base64Url-encode the calculated signature and compare it securely to the provided signature component. Return true if they match.
    Per D-03 (cancelling AUTH-03), do not check or enforce expiration timestamps.
    Per D-04 (cancelling AUTH-04), do not bind or check IP addresses.
    Remove `session_destroy()` logic, making it an empty stub (as tokens are stateless and cannot be destroyed server-side in Phase 1).
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
