# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint below was discovered through failure.
> Acknowledge each one explicitly before proceeding.

- [ ] CONSTRAINT: mbedtls HMAC Signature Junk — Failing to properly initialize/check `mbedtls_md_setup` or associated crypto libraries on ESP32 results in uninitialized stack memory junk (e.g. `CECMwQhAhM_8PwgAAADG____AAD_AAAAAP8AAAAAAAA`) being generated as the signature instead of actual hash bytes. — If stateless JWT is to be used, mbedtls return codes must be strictly checked and verified, or an alternative session store used.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Unchecked mbedtls calls | Calling mbedtls setup/starts/update/finish without verifying return codes | blocking | Assert on return value of `mbedtls_md_setup`, `mbedtls_md_hmac_starts` etc. or use in-memory slot-based session tokens. |
