<task type="auto">
  <name>Task 2: Implement Staging APIs</name>
  <files>components/nvs_store/nvs_store.c, components/nvs_store/nvs_store.h</files>
  <action>
    Add a staging namespace/keys mechanism for Wi-Fi and credentials (per RECOV-03).
    1. Add `staging_type` key (0=none, 1=wifi, 2=credentials) in NVS.
    2. Add `nvs_store_stage_wifi(const char* ssid, const char* pass)`: saves values with a `stg_` prefix (e.g. `stg_ssid`, `stg_pass`) and sets `staging_type = 1`.
    3. Add `nvs_store_stage_credentials(const char* user, const char* pass)`: saves values with a `stg_` prefix and sets `staging_type = 2`.
    4. Update getters (`nvs_store_get_wifi_config`, `nvs_store_get_credentials`): check `staging_type`. If it matches what they are retrieving, return the `stg_` prefixed keys; otherwise, return the normal keys.
    5. Add `nvs_store_commit()`: reads `staging_type`. If 1 or 2, copy `stg_` values to normal keys, then erase `stg_` keys and set `staging_type = 0`.
    6. Add `nvs_store_rollback()`: erase `stg_` keys and set `staging_type = 0`.
    7. Add `uint8_t nvs_store_get_staging_type(void)` to allow checking staging state on boot.
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>Changes to Wi-Fi or credentials can be staged, committed, or rolled back safely using NVS.</done>
</task>
