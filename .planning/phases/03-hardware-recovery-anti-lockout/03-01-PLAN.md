<task type="auto">
  <name>Task 1: Implement SoftAP Timeout and Client Awareness</name>
  <files>components/wifi_manager/wifi_manager.c, components/wifi_manager/wifi_manager.h, components/app_config/app_config.h</files>
  <action>
    Modify `wifi_manager` to support delayed SoftAP shutdown (per D-08, D-09, RECOV-02).
    1. Define `APP_CONFIG_AP_TIMEOUT_MS` (600000 = 10 mins) in `app_config.h`.
    2. Remove any logic that automatically opens SoftAP on STA failure.
    3. Add public API `wifi_manager_start_recovery_ap()`. When called, it should ensure SoftAP is running and start/reset a 10-minute FreeRTOS timer (`s_ap_timeout_timer`).
    4. In the timer callback, use `esp_wifi_ap_get_sta_list()`. If `num > 0`, restart the timer (pause timeout). If `num == 0`, shut down the SoftAP.
    5. Add public API `bool wifi_manager_is_ap_active(void)` to allow polling the AP state for LED feedback.
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>SoftAP requires manual invocation and shuts down automatically if no clients are connected for 10 minutes.</done>
</task>

<task type="auto">
  <name>Task 2: Implement 30s Staged IP Check</name>
  <files>components/wifi_manager/wifi_manager.c, components/wifi_manager/wifi_manager.h</files>
  <action>
    Add a mechanism to rollback if an IP isn't acquired quickly after setting new Wi-Fi credentials (per D-10, D-11, RECOV-03).
    1. Add public API `wifi_manager_begin_staged_ip_check()`.
    2. When called, it sets a local flag `s_staged_ip_check = true` and starts a 30-second FreeRTOS timer.
    3. Modify `WIFI_EVENT_STA_GOT_IP` event handler (or similar): if `s_staged_ip_check` is true, call `nvs_store_commit()`, stop the 30s timer, and clear the flag.
    4. In the 30s timer callback: if it fires (meaning no IP was acquired), call `nvs_store_rollback()` and immediately call `esp_restart()` to revert to the old Wi-Fi settings.
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>If `wifi_manager_begin_staged_ip_check()` is called, the system enforces a 30-second deadline to get an IP or reboots.</done>
</task>
