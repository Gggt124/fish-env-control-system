<task type="auto">
  <name>Task 1: Hardware UI Task</name>
  <files>main/recovery_task.c, main/recovery_task.h, components/app_config/app_config.h, main/CMakeLists.txt</files>
  <action>
    Create a FreeRTOS task to poll buttons and drive LED feedback (per D-01 through D-07).
    1. Define GPIOs in `app_config.h`: `APP_CONFIG_BOOT_BTN_GPIO` (0), `APP_CONFIG_EXT_BTN_GPIO` (4), `APP_CONFIG_BOARD_LED_GPIO` (2), `APP_CONFIG_EXT_LED_GPIO` (5). (Use standard constants).
    2. Create `main/recovery_task.c/h` and add to `main/CMakeLists.txt` `SRCS`.
    3. Implement `recovery_task_start()` which configures the GPIOs (inputs with internal pull-up, outputs) and spawns a FreeRTOS task.
    4. Task logic (poll every 50ms):
       - Mutual Exclusion: if one button is pressed, ignore the other (D-02).
       - Track press duration. 
       - LED State: 0-2s = Solid ON, 2-5s = Slow Blink, >5s = Fast Blink (D-06). Both LEDs reflect this (D-05).
    5. On button release:
       - If duration >= 5s: call `nvs_store_factory_reset_credentials()` then `wifi_manager_start_recovery_ap()` (D-04).
       - If duration 2s-5s: call `wifi_manager_start_recovery_ap()` (D-03).
    6. When not pressed, keep LED Solid ON if `wifi_manager_is_ap_active()` returns true, else turn LED OFF (D-07).
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>A polling task reliably handles button debouncing, press duration, and coordinates LEDs with AP state.</done>
</task>

<task type="auto">
  <name>Task 2: Boot Coordination & Rollback Timers</name>
  <files>main/app_main.c</files>
  <action>
    Hook the hardware task into the boot process and handle unconfirmed staging (per RECOV-03, D-10, D-11).
    1. In `app_main.c` `app_main()`, call `recovery_task_start()` early on.
    2. Check `nvs_store_get_staging_type()`.
    3. If type == 1 (Wi-Fi): Call `wifi_manager_begin_staged_ip_check()` (from Plan 01) so Wi-Fi rolls back if no IP is acquired in 30s.
    4. If type == 2 (Credentials): Create and start a 3-minute FreeRTOS timer. In the timer callback, call `nvs_store_rollback()` and `esp_restart()`. This acts as the anti-lockout if `/api/confirm` is never called.
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>Boot sequence correctly delegates staging rollback checks and initializes hardware recovery features.</done>
