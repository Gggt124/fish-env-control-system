<objective>
Update HTTP endpoints to leverage the NVS staging mechanism instead of applying changes immediately.

Purpose: Protects the device from getting permanently disconnected if a user inputs an invalid Wi-Fi password or forgets their newly set admin password.
Output: Updated `/api/wifi/connect` and credential update handlers, plus a new `/api/confirm` endpoint.
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Stage Updates and Reboot</name>
  <files>main/web_server.c</files>
  <action>
    Change Wi-Fi and credential update APIs to use the new staging system (per RECOV-03).
    1. In `/api/wifi/connect` (or its equivalent handler): Instead of saving directly and restarting Wi-Fi, call `nvs_store_stage_wifi()`, return a 200 OK JSON response, and trigger `esp_restart()` via a short FreeRTOS timer (e.g., 500ms) to allow the HTTP response to complete.
    2. In the credential update handler (added in Phase 2): Instead of saving directly, call `nvs_store_stage_credentials()`, return a 200 OK response, and trigger `esp_restart()` via a 500ms timer.
    Note: Wait for the HTTP response to be fully sent before rebooting.
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>Config changes correctly stage data and cleanly reboot the device.</done>
</task>

<task type="auto">
  <name>Task 2: Implement Confirm API</name>
  <files>main/web_server.c, main/web_server.h</files>
  <action>
    Provide an endpoint to commit staged credentials/settings (per RECOV-03).
    1. Add a POST handler for `/api/confirm`.
    2. Secure this endpoint with the authentication middleware (it must require a valid session).
    3. The handler should call `nvs_store_commit()` and return `{"ok":true}`.
    4. Register this handler in the HTTP server initialization.
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>A protected `/api/confirm` endpoint exists and commits NVS staged changes.</done>
</task>
