## Implementation Decisions

### IP Binding Validation (AUTH-04)
- **D-01:** Remove the IP validation check (`strcmp(s_slots[i].ip, client_ip) == 0`) inside [session_validate()](file:///C:/Users/Copter/.gemini/antigravity/worktrees/fish-pump-RelayTimerControl/review-gsd-plan-convergence/components/session/session.c#L100-L123) in order to satisfy AUTH-04 and prevent session disconnects during interface fallback (SoftAP/STA). Retain the `client_ip` parameter in the signature for compatibility.

### Global Session Invalidation (AUTH-06)
- **D-02:** Implement a new function `session_invalidate_others(const char *current_token)` in [session.c](file:///C:/Users/Copter/.gemini/antigravity/worktrees/fish-pump-RelayTimerControl/review-gsd-plan-convergence/components/session/session.c) to invalidate all other active sessions except the currently active one.
- **D-03:** Invoke `session_invalidate_others()` in [handle_api_confirm_post()](file:///C:/Users/Copter/.gemini/antigravity/worktrees/fish-pump-RelayTimerControl/review-gsd-plan-convergence/main/web_server.c#L3194-L3209) when credential changes (type=2) are confirmed. This satisfies global invalidation on password change without forcing a double-redirect login bounce for the user who is confirming the change.

### Credentials Staging & NVS Integrity (RECOV-03 & Integration)
- **D-04:** Fix [nvs_store_commit_staging()](file:///C:/Users/Copter/.gemini/antigravity/worktrees/fish-pump-RelayTimerControl/review-gsd-plan-convergence/components/nvs_store/nvs_store.c#L1229-L1277) to call `nvs_commit(handle)` after writing the staged credentials (`admin_user` and `admin_pass`) for type=2, resolving the silent write discard bug.
- **D-05:** Wi-Fi connection and profile saves will intentionally bypass the staging and confirmation banner mechanism, making updates instant and direct as specified by the user's latest design choice. Only credentials change (type=2) uses the staging, reboot, and confirmation flow.

### Phase 2 Verification Report
- **D-06:** Create a formal verification report `02-VERIFICATION.md` under `.planning/phases/02-credential-management-ui-logic/` to address the missing verification report audit gap.
