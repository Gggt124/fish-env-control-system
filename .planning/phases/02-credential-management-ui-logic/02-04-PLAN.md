# Plan 04: Revert JWT and Fix Rate Limiter

## Objective
Revert the session mechanism from JWT to a simple 4-slot in-memory token array using `esp_random`, and fix the login rate limiter eviction strategy.

## Context
- The user requested the removal of JWT (previously implemented in Phase 2) and reverting to the original `AGENTS.md` specification of `esp_random` 4 slots.
- The `gsd-code-reviewer` identified a logic flaw in the rate limiter eviction strategy (WR-01) where index 0 is always evicted when unblocked, rendering LRU ineffective.

## Tasks

<task type="auto" id="revert_session_h">

  <files>

  </files>
  <implementation>

    - Keep `session_init`, `session_create`, `session_validate`, `session_destroy`.

  </implementation>
</task>

<task type="auto" id="revert_session_c">
  <behavior>Rewrite `components/session/session.c` to use an array of 4 slots for sessions.</behavior>
  <files>
    - components/session/session.c
  </files>
  <implementation>
    - Define a struct `session_slot_t` containing `char token[65]` and `char ip[64]`, plus an active flag and `int64_t last_used` timestamp.
    - Define an array of 4 slots, protected by `s_session_mutex`.
    - `session_init` initializes the array (sets active to false) and creates the mutex.
    - `session_create` generates 32 random bytes using `esp_random()`, converts to a hex string. It locks the mutex, finds an empty slot or replaces the oldest slot (based on `last_used`), stores the hex string and client_ip, updates `last_used`, and sets active=true. Returns the hex string in `token_out`.
    - `session_validate` locks mutex, iterates through the 4 slots. If a slot is active, token matches, and client_ip matches, update `last_used` and return true. Otherwise false.
    - `session_destroy` locks mutex, finds the matching token and sets active=false.
    - `session_invalidate_all` locks mutex, iterates through all 4 slots and sets active=false.
    - Remove all mbedtls code, base64 url encoding, and HMAC NVS loading code.
  </implementation>
</task>

<task type="auto" id="update_web_server_session_calls">

<task type="auto" id="fix_rate_limiter">
  <behavior>Fix rate limiter eviction strategy in `main/web_server.c` by adding a `last_seen` timestamp.</behavior>
  <files>
    - main/web_server.c
  </files>
  <implementation>
    - Update `login_rate_limit_t` struct to include `int64_t last_seen`.
    - In `handle_api_login`, update the eviction loop to evict the slot with the oldest `last_seen` rather than the minimum `block_until`.
    - Ensure `last_seen` is updated on every login attempt for that IP.
  </implementation>
</task>

## Success Criteria
- JWT components (mbedtls) are completely removed from the session module.
- `session.c` implements a 4-slot array with `esp_random()`.
- Password change correctly invalidates all active sessions.
- Rate limiter correctly evicts the least recently seen IP when full.
- `idf.py build` succeeds.
