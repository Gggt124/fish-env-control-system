<task type="auto">
  <name>Task 2: Add Credentials Update Route</name>
  <files>main/web_server.c</files>
  <action>Add a new HTTP POST handler `handle_api_auth_credentials` (route `/api/auth/credentials`). This route MUST be protected by session auth and `is_same_origin`.
It extracts `nonce`, `current_password`, `new_username`, and `new_password` from the JSON body.
Validation logic:
1. Reject if `s_auth_nonce` is empty or does not match `nonce`.
2. Clear `s_auth_nonce` immediately after checking to prevent replay (AUTH-07).
3. Fetch current credentials via `nvs_store_get_credentials` and verify `current_password` matches.
4. If `new_username` is empty or omitted, fallback to the `current_username` obtained in step 3.
5. Save new credentials using `nvs_store_set_credentials`.
6. Call `session_regenerate_secret()` to invalidate all existing sessions (AUTH-06, D-03).
7. Return `{"ok":true}`.
Register the URI in `start_webserver()`.
Per D-02, D-03.</action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>Credentials POST endpoint works and safely updates credentials.</done>
