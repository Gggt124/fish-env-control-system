# Phase 02 Gap Closure Plan

## Objective
Fix the issue where entering an incorrect current password in the Change Password modal gets stuck in a loading state and redirects to the login page.

## Root Cause
When the current password does not match, the backend returns `401 Unauthorized`. The frontend has a global interceptor that catches any 401 response and immediately logs the user out and redirects to `/login`, bypassing the modal's error handling.

## Proposed Changes

### `main/web_server.c`
- In `handle_api_auth_credentials()`, change the response for an incorrect current password from `"401 Unauthorized"` to `"400 Bad Request"`. This prevents the global interceptor from triggering and allows the frontend to process the `{"ok":false,"error":"invalid_credentials"}` response.
