<task type="auto">
  <name>Task 1: Replace absolute test deadline with duration-based countdown logic</name>
  <files>components/cooling_control/cooling_control.c</files>
  <action>
Modify components/cooling_control/cooling_control.c to:
1. Replace `s_test_deadline_ms` with two new static variables: `s_test_remaining_ms` (to track remaining runtime in milliseconds) and `s_last_test_update_ms` (to track the last update timestamp).
2. Update `reset_runtime_state_locked()` to initialize `s_test_remaining_ms = 0;` and `s_last_test_update_ms = 0;` (removing `s_test_deadline_ms`).
3. Update `test_remaining_locked()` to calculate and return the remaining test time in seconds derived from `s_test_remaining_ms`. If the mode is not `COOLING_CONTROL_MODE_TEST_ON`, return 0.
4. Update `cooling_control_start_test()` to initialize `s_test_remaining_ms` to `(int64_t)s_config.test_timeout_sec * MS_PER_SEC` and `s_last_test_update_ms` to `now_ms`.
5. Update `cooling_control_apply_config()`, `cooling_control_stop()`, and `cooling_control_set_mode()` to clear `s_test_remaining_ms` and `s_last_test_update_ms` to 0 when stopping or leaving Test ON mode.
6. In `update_control_locked()`, update the Test ON countdown block:
   - Calculate elapsed time `elapsed_ms = now_ms - s_last_test_update_ms;` (with a floor of 0).
   - Determine if the compressor protection lockout is currently active by checking if `lockout_remaining_locked(now_ms) > 0`.
   - If lockout is NOT active, subtract `elapsed_ms` from `s_test_remaining_ms`.
   - If `s_test_remaining_ms <= 0`, set it to 0 and switch `s_mode` to `s_previous_mode`.
   - Always update `s_last_test_update_ms = now_ms;` at the end of the Test ON update block to ensure elapsed time is accurately tracked.
  </action>
