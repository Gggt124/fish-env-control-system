<task type="auto">
  <name>Task 1: Fix TFT UI Formatting and Logic Gaps</name>
  <files>main/tft_display.c</files>
  <action>
    Fix two TFT UI issues in `main/tft_display.c`:
    
    1. **Wi-Fi Formatting:** Increase the format specifier width for `wifi_formatted` from 13 to 15.
       - Change `snprintf(wifi_formatted, sizeof(wifi_formatted), "%-13.13s", wifi_str);` to `%-15.15s` (or simply `%-15s` if appropriate, but ensure it uses the full 15 chars).
       - Ensure `wifi_formatted` buffer is `char wifi_formatted[16]`. It should already be correctly sized, but verify it.
       - Gap reason: The %-13.13s limitation cuts off strings longer than 13 characters like 'AP:192.168.4.1'.
    
    2. **Pump Phase Display Logic:** Override the Phase display string when the pump is not running.
       - In the "5. Pump Phase" section, update the logic to check `if (!pump_running)`. If the pump is NOT running, set `phase_name = "-"` (or "IDLE") and `phase_color = TFT_COLOR_GRAY`, regardless of the `pump.phase` value.
       - Gap reason: When `pump_running` is false, `pump.phase` might still reflect the active timer's current cycle, causing the TFT to show "ON" while the relay is completely stopped.
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>Long IP addresses are no longer truncated on the TFT and Pump Phase correctly shows IDLE/- when pump is stopped.</done>
</task>
