<task type="auto">
  <name>Task 2: Implement periodic update task with state-cached differential rendering</name>
  <files>main/tft_display.h, main/tft_display.c, main/app_main.c</files>
  <action>
    - Implement a FreeRTOS task 'tft_display_task' in main/tft_display.c that runs continuously.
    - Fetch system states periodically (every 500ms for countdown and float, or immediately upon transition):
      - Call pump_control_get_status() for pump state, active timer, phase, and countdown seconds.
      - Call cooling_control_get_status() for temperature, cooling relay state, cooling mode, and lockout.
      - Call wifi_manager APIs for AP/STA state and current IP.
    - Cache the status values locally in static variables inside the task.
    - Perform differential updates: compare current state values with cached ones and only redraw changed fields (with font background clearing) to avoid flickers and save SPI traffic.
    - Apply HSL color palette principles to dynamic states (Green for Running/ON, Gray for Stopped/OFF, Red/Yellow for Faults/Lockouts).
    - Declare 'void tft_display_start_task(void);' in tft_display.h and implement it in tft_display.c to spawn the task via xTaskCreate() with stack size 3072 and priority 4 (low priority to protect main loop).
    - In main/app_main.c, replace the temporary booting splash clear with a call to tft_display_start_task() after all components (including Wi-Fi and Web server) initialize.
  </action>
  <verify>
    <automated>idf.py build</automated>
  </verify>
  <done>
    FreeRTOS update task is implemented with differential rendering and started on boot.
  </done>
</task>
