  <name>Task 1: Implement UI logic refinements in client-side JS</name>
  <files>main/static/app.js</files>
  <action>
Modify main/static/app.js to:
1. Wrap the API logout request in `doLogout()` with a `confirm("คุณต้องการออกจากระบบใช่หรือไม่?")` check. If rejected, return early (per D-15-07 / UI-23).
2. Wrap the API disconnect request in `doDisconnect()` with a `confirm("คุณต้องการตัดการเชื่อมต่อ Wi-Fi ใช่หรือไม่?")` check. If rejected, return early (per D-15-04 / UI-20). Change the disconnect button text to "Disconnecting..." when clicked, and restore/update to "Disconnect" on success/failure toast messages (per D-15-03 / UI-19).
3. Implement `checkPumpDirty()` to compare currently entered values in pump config form inputs against `pumpConfig` baseline and return `true` if any value differs. Update `markPumpDirty()` to check this function, toggle the warning display `pump-unsaved-warning` (show if dirty, hide if clean), and toggle `pumpDirty` variable. Update `updatePumpConfigSaveButton()` to disable the save button if not dirty (per D-15-05, D-15-06 / UI-21, UI-22).
4. Implement `checkCoolingDirty()` to compare currently entered values in cooling config form inputs against `coolingConfig` baseline and return `true` if any value differs. Update `markCoolingDirty()` to check this function, toggle the warning display `cooling-unsaved-warning` (show if dirty, hide if clean), and toggle `coolingDirty` variable. Update `updateCoolingConfigSaveButton()` to disable the save button if not dirty (per D-15-05, D-15-06 / UI-21, UI-22).
5. Implement `checkHardwareDirty()` to compare hardware dropdown select values against baseline active/pending mapping and return `true` if any select differs. Update `markHardwareDirty()` to check this function and toggle the unsaved warning `hardware-unsaved-warning` (show if dirty, hide if clean) and `hardwareDirty` variable (per D-15-02 / UI-18).
6. Update `renderHardwareSummary()` to accept a fourth parameter `activeMap`. If `activeMap` is provided (during pending summary rendering), compare the pending mapping fields with active mapping fields. If a field matches the active mapping, do not render its row. Count the number of changes rendered. If count is 0, render a clean empty label: "No pending changes" (per D-15-01 / UI-17).
7. In `updateCoolingButtons()`, disable cooling mode action buttons if the device is already in that active mode (e.g. disable `cooling-mode-auto` if current mode is `auto`) to match existing dashboard action logic (per REG-01).
