<objective>
Extend `components/pump_control` from a single-relay selected-timer runtime to a safe two-channel pump runtime.

Purpose: Make the core firmware capable of driving Relay 1 and Relay 2 independently according to one binary float input, with start-phase semantics, stopped-state float preview, and fail-safe relay handling before boot/API/UI layers consume the new contract.
Output: `pump_control` exposes a dual-relay config/status contract and implements Float ON -> Timer 1/Relay 1, Float OFF -> Timer 2/Relay 2, confirmed-transition restart behavior, stopped preview, and both-relays-off safety.
</objective>
