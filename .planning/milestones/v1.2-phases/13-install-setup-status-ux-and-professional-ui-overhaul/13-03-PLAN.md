# Phase 13, Plan 3: Hardware/Install & Status UX (Side-by-side Bento & Text Wrap)

## Goal
Implement side-by-side Bento cards for Active vs. Pending GPIO configurations (D-05), ensure text wrapping for long diagnostics on the Status page (D-09), and explicitly document the DS18B20 pull-up requirement on the Hardware page.

## Steps

1.  **Hardware/Install Layout (Active vs. Pending):**
    - Open `main/static/hardware.html`.
    - Modify the `.hardware-summary-grid` to visually separate the "Active GPIO Map" and "Pending Reboot Map" into side-by-side (or stacked on mobile) Bento cards.
    - Apply a success-green theme (`var(--secondary)` / `.good`) border/badge to the Active Map.
    - Apply a warning-orange theme (`var(--tertiary)` / `.warn`) border/badge to the Pending Reboot Map.
    - Explicitly add the 4.7 kOhm DS18B20 pull-up warning near the DS18B20 selector so installers know it's a hardware requirement.

2.  **Status Page Text Wrap Proofing:**
    - Open `main/static/status.html`.
    - Add CSS classes or inline styles (e.g., `word-break: break-all;` or `overflow-wrap: anywhere;`) to the `.info-value` elements.
    - Ensure long SSID, IP, MAC, and Uptime values do not break the Bento Grid boundaries and wrap correctly on narrow viewports (`375px`).

3.  **Final Polish:**
    - Test all responsive grid layouts (Dashboard, Hardware, Status, Wi-Fi) under the new Dark Glassmorphism Bento Grid.
    - Ensure Thai language recovery texts are legible against the dark background.

## Verification
- Resize the browser to `375px` and verify that Status text wraps properly without scrolling horizontally.
- On the Hardware page, verify that modifying a GPIO selector clearly highlights the Pending Reboot Map in orange, contrasting with the Active Map in green.
