<task type="auto">
  <name>Task 1: Loading State and Modal CSS Base</name>
  <files>main/static/index.html, main/static/style.css</files>
  <action>
  In `main/static/index.html`, append `<div id="modal-container" class="modal-overlay hidden">` right before `</body>`. Inside it, include a `.modal-card` with an `h3` for title (`id="modal-title"`), `p` for message (`id="modal-message"`), and a `.modal-actions` div containing a Cancel `.btn.btn-outline` (`id="modal-cancel"`) and a Confirm `.btn.btn-primary` (`id="modal-confirm"`).
  
  In `main/static/style.css`:
  - Define `--spinner-color: currentColor;` on `.btn` and add `transition: background-color 0.2s, opacity 0.2s, color 0.2s;`.
  - Override `--spinner-color` for `.btn-primary` and `.btn-danger` to use `var(--on-primary)` and `var(--on-error)` respectively.
  - Create `.btn.loading` that sets `color: transparent !important` and `pointer-events: none`.
  - Create `.btn.loading::after` that renders an absolute-centered CSS spinner using `--spinner-color` as the top border color.
  - Add `.modal-overlay` styling with `position: fixed`, `background: rgba(0,0,0,0.4)`, and `backdrop-filter: blur(4px)` for glassmorphism. Make it hidden with opacity 0 and `pointer-events: none`.
  - Style `.modal-card` with `var(--surface)`, border-radius, shadow, and a subtle transform animation.
  </action>
  <verify>
    <automated>grep -c "id=\"modal-container\"" main/static/index.html</automated>
  </verify>
  <done>CSS handles `.loading` state seamlessly and modal markup is injected.</done>
</task>

<task type="auto">
  <name>Task 2: JS Loading & Modal Integration</name>
  <files>main/static/app.js</files>
  <action>
  Create a global `setLoading(btnId_or_element, isLoading)` function that toggles the `.loading` class and `disabled` property on buttons.
  
  Replace all occurrences of manual button spinner injection (`btn.innerHTML = '<span class="spinner"></span>...';`) and text replacement (`btn.textContent = '...';`) during API calls with `setLoading(btn, true)` and `setLoading(btn, false)`. Target `login`, `doScan`, `doConnect`, `doDisconnect`, `savePumpConfig`, `saveCoolingConfig`, and `applyHardwareMap` handlers. Remove inline text modifications for these buttons.
  
  Create `showConfirmModal(title, message, onConfirm)` function that populates the `#modal-container` title and text, sets up click handlers for Cancel and Confirm, shows the modal by removing the `.hidden` class, and executes the callback upon confirmation while cleaning up event listeners and hiding the modal.
  
  Replace native `confirm()` calls in `doLogout()` and `doDisconnect()` with `showConfirmModal()`.
  </action>
  <verify>
    <automated>grep -c "showConfirmModal(" main/static/app.js</automated>
  </verify>
  <done>All `confirm()` prompts are replaced, and buttons use `.loading` states.</done>
</task>
