# Milestone v1.7: Authentication & Recovery

**Status:** ✅ SHIPPED 2026-06-14
**Phases:** 1–3.1
**Total Plans:** 14

## Overview

Milestone v1.7 improves access control for the Fish Pump Relay Timer Control firmware by adding persistent sessions, user credential management, and physical hardware recovery fail-safes. The firmware now supports "Remember Me" persistent login, a full credential change flow with anti-replay nonces, and GPIO-triggered SoftAP fallback and factory reset — all without losing Wi-Fi or pump configurations.

## Milestone Summary

**Decimal Phases:**

- Phase 03.1: Integration Remediations (inserted after Phase 3 to close audit gaps RECOV-03, AUTH-04, AUTH-06)

**Key Decisions:**

- Use stateful in-memory slot-based session store instead of stateless JWT — mbedtls HMAC-SHA256 produced signature mismatches due to uninitialized memory stack junk, blocking login flow.
- Remove IP binding from session validation — ensures connectivity is not dropped during SoftAP/STA fallback routing and offline operations (AUTH-04).
- Implement `session_invalidate_others()` to keep the credential-changing user authenticated while logging out all other devices (AUTH-06).
- Add `nvs_commit()` to staging credential verification — ensures credential changes survive boot cycles (RECOV-03).
- SoftAP fallback remains active during staging validation to prevent locking out the user during the 3-minute rollback window.
- Replace blocking `alert()` calls with non-blocking `showToast()` for confirmation/rollback flows.
- Skip staging for Wi-Fi profiles that match existing credentials in NVS.

**Issues Resolved:**

- mbedtls HMAC-SHA256 signature mismatch blocking login — pivoted to stateful slot-based session store.
- Session dropout during SoftAP/STA routing transitions — removed IP binding from session validation.
- Credential changes not surviving reboots — added `nvs_commit()` to staging confirmation path.
- All other active sessions not being invalidated on password change — implemented targeted `session_invalidate_others()`.
- Missing Phase 2 verification report — created `02-VERIFICATION.md`.

**Issues Deferred:**

- Orphaned `nvs_store_stage_wifi` export in `components/nvs_store/nvs_store.c` — unused because Wi-Fi connects live before save. No functional impact.

**Technical Debt Incurred:**

- `nvs_store_stage_wifi` function exported but unused — can be removed in a cleanup pass.
